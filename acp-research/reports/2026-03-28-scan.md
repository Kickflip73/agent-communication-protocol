### 关键发现

#### 1. OIXA Protocol 加入 A2A 生态（2026-03-27，#1692）

OIXA 是基于 A2A 构建的 Agent 自主经济协议：

- **定位**：AI Agent 自主雇佣 + 支付层，构建于 A2A 通信层之上
- **协议栈**：A2A + MCP（通信）/ x402 Protocol（HTTP 自主支付）/ Base Mainnet + USDC（链上托管）/ ERC-8004（链上 Agent 身份注册）
- **核心机制**：
  - 反向拍卖发现：agents 出价竞争任务，最低出价胜出
  - 链上 USDC 托管：bid 接受时锁仓，验证交付后自动释放
  - 密码学输出验证：不只是交付确认，还有质量验证
  - 20% 质押防 Sybil 攻击
  - 链上声誉积累（每次 A2A 交易构建可验证分数）
- **SDK**：`pip install oixa-protocol`（支持 LangChain、CrewAI、AutoGen、Haystack、MCP）
- **Agent Card**：`http://64.23.235.34:8000/.well-known/agent.json`

#### 2. Python Tutorial 更新至 a2a-sdk v1.0.0-alpha.0（2026-03-26，#1678）

Schema 层面的重要变更：

- **AgentCard schema 更新**：`url` 属性 → `icon_url`（Breaking Change for v0.x users）
- **新增 `supported_interfaces` 字段**：协议绑定声明（声明 Agent 支持的接口协议）
- **新增 `extended_agent_card` 能力**
- **引入 `ClientFactory`**：替代直接 `A2ACardResolver` 实例化（封装工厂模式）

- **安全修复**：Server 绑定从 `0.0.0.0` 改为 `127.0.0.1`（本地测试安全默认值）
- **TaskStatusUpdateEvent 语义明确化**：区分 `TASK_STATE_WORKING` → `TASK_STATE_COMPLETED` 事件序列

## 对 ACP 的启示

### 立即可同步项

| 优先级 | 内容 | 说明 |
|--------|------|------|
| **P2** | AgentCard `limitations` 字段 | 参考 A2A #1694，在 AgentCard schema 中加可选 `limitations: string[]`，与 `capabilities` / `availability` 构成"三元组"（能做什么 / 何时活跃 / 不能做什么） |
| **P3** | trust.signals[] 信任信号设计 | A2A #1628 的 4 信号分类（behavioral/vouch_chain/onchain_credentials/onchain_activity）中，`behavioral`（运行时成功率、策略合规性）对 ACP 有参考价值。可在 v2.x 阶段评估引入轻量行为信任信号 |
| **INFO** | A2A `supported_interfaces` 字段 | 与 ACP `capabilities` 字段定位相同，ACP 实现更完整，无需同步，保持领先优势 |
| **INFO** | OIXA 经济协议监控 | A2A 生态开始出现支付/经济层，ACP 短期无需跟进（与轻量原则相悖），但可预留 `payment_endpoints` 可选扩展点 |

### ACP 当前竞争优势确认

1. **Identity**：ACP `did:acp:` + 自动验证（v1.8/v1.9）领先 A2A #1672（70 评论，仍无官方结论）
2. **Capability 声明**：ACP `capabilities + extensions + availability` 优于 A2A `supported_interfaces`（alpha）
3. **Offline 投递**：ACP v2.0 offline queue 无 A2A 对应实现
4. **Spec 迭代速度**：ACP 持续合并，A2A 代码冻结第 12 天

### 不需要同步的内容

- ANP 协议变更：无
- A2A SDK 工厂模式（`ClientFactory`）：纯实现细节，与 ACP spec 无关
- OIXA 链上经济层：超出 ACP 定位范围
