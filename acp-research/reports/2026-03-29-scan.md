### Issue #1694 — Add `limitations` field to AgentCard（2026-03-27）
**提案摘要**：AgentCard 增加可选 `limitations` 字段，用于描述 Agent **不能做什么**（语言限制、文件大小上限、领域限制）。
**动机**：当前 AgentCard 只说能做什么（skills/capabilities），无法表达边界。Orchestrator 需要知道 limitations 才能做出更好的路由决策。
**ACP 机会**：可在 v2.15 快速实现 — AgentCard 的 `capabilities` 里加 `limitations: {}` dict，先于 A2A 落地。字段可复用 trust.signals[] 的模式。

### PR #1696 — identity verification guidance for agent discovery（2026-03-28）
**内容**：给 agent-discovery.md 加"验证 Agent 身份"章节 + Future Considerations（registry 标准化、能力+身份组合、委托链）。
**ACP 现状**：ACP v0.8/v1.3/v1.8/v1.9 已实现完整 Ed25519+DID+AgentCard 签名+对端自动验证链。**我们领先 A2A 整整一个文档周期**。
**行动**：在 ACP README 中补充一节「与 A2A trust 对比」，凸显差异化。

### Issue #1690 — Clarify tenant transmission mechanism（open）
多租户/租户透传机制，企业级功能。**ACP 不做**，符合「永远不做 OAuth 2.0 + 多租户」定位。
