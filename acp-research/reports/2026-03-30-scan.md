### Hot Issues / PRs (2026-03-29)

#### PR#1696 — docs: add identity verification guidance for agent discovery
- **State**: Open (2026-03-28)
- **Key points**:
  - 新增 "Verifying Agent Identity" 章节到 agent-discovery.md
  - Future Considerations 里明确列出 **delegation chains** 为待探索方向
  - 引用 JWS (RFC 7515) + JSON Canonicalization (RFC 8785)
- **🎯 ACP 差异化机会**: ACP v2.16.0 已实现 delegation_chain（Ed25519签名，zero-registry验证）。A2A 仍在讨论阶段。

#### IS#1672 — Proposal: Agent Identity Verification for Agent Cards
- **73+ 评论**，社区高度关注
- 核心问题：A2A AgentCard 无协议级身份验证机制，依赖 TLS/OAuth（authorization ≠ identity）
- **🎯 ACP 优势**: ACP v0.8+ 已有 Ed25519 keypair + self-sovereign DID。AgentCard 自签名（v1.8）。

#### IS#1667 — Heartbeat-based agents: availability metadata and offline-first task handling
- A2A 假设 agent 是持续在线的服务，无法处理「定时唤醒」的 heartbeat agent
- 提案：AgentCard 增加 availability_metadata（schedule、next_available_at 等）
- **🎯 ACP 机会**: 可在 AgentCard 中增加 `availability.schedule` 字段，支持离线队列 flush（ACP 已有 offline_queue）

#### IS#1694 — Add 'limitations' field to AgentCard
- 提案：AgentCard 增加 `limitations[]` 字段，描述 agent 不擅长做什么
- 用于 orchestrator 路由决策
- **🎯 ACP 机会**: 未来版本可在 AgentCard 中添加 `limitations` 字段

#### IS#1029 — Support publish/subscribe methods [TSC Review]
- Pub/Sub 模式，目前 A2A 是纯 request/response + SSE streaming
- TSC 审核中，尚无结论
- **🎯 ACP 现状**: ACP 有 SSE streaming，未实现 pub/sub。可关注后续

## 战略评估

### ACP 当前领先点
1. **delegation_chain (v2.16)** — A2A 在 Future Considerations，ACP 已发布
2. **Ed25519 zero-registry 身份** — A2A IS#1672 还在讨论，ACP v0.8 已有

4. **offline_queue + flush on reconnect** — A2A IS#1667 在讨论，ACP 已实现

### 下一版本建议 (v2.17)
**候选方向**：
- `availability.schedule` —— AgentCard 增加心跳/定时 agent 的可用时间窗口元数据（对应 A2A IS#1667）

- Pub/Sub 轻量支持 —— 基于现有 SSE 实现 topic 订阅（对应 A2A IS#1029）

**推荐**: `availability.schedule` — 差异化最强，且 ACP 的离线队列基础已就绪，实现成本低。
