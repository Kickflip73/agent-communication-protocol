### Issue 追踪更新

#### #1681 — GetTaskPushNotificationConfig 返回凭证明文（安全漏洞）
- **状态**: ⚠️ open，**无官方回应**
- **问题核心**: `PushNotificationConfig` 对象包含认证凭证（credentials），`GetTaskPushNotificationConfig` / `ListTaskPushNotificationConfigs` API 直接返回，造成敏感信息泄露
- **影响**: 任何调用该 API 的客户端都会获取到推送配置的认证信息
- **ACP 优势**: ACP 无 PushNotification 系统，设计禁忌之一，**从根本上规避此类漏洞**

#### #1680 — Cancel Task 无法立即完成时如何响应
- **状态**: ⚠️ open，社区讨论中，**A2A 官方无结论**
- **当前讨论的两种方案**:
  - 方案 A：同步返回 WORKING 状态 + 完成后 webhook 回调
  - 方案 B：（截断，未见）
- **问题根源**: A2A cancel 设计为 async-capable，但 spec 未明确异步完成路径
- **ACP 优势**: v1.5.2 已明确为**同步即时**（`:cancel` 一次调用即完成），幂等，`ERR_TASK_NOT_CANCELABLE` 409 处理 terminal 状态。设计简单，无歧义
