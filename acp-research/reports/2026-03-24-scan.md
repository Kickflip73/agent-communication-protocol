### 关键 Open Issues（截至 2026-03-23）

#### #1672 — Agent Identity Verification（身份验证提案）
- **内容**：AgentCard 缺乏密码学身份验证，没有标准方式验证"我在和谁通信"
- **意义**：与我们的 `did:acp:` 方向高度一致。A2A 还在讨论阶段，我们 v1.3 已实现
- **ACP 优势**：`did:acp:` Ed25519 key-derived DID，自主身份，无注册中心；A2A 仍在 proposal 阶段

#### #1667 — Heartbeat-based Agents（非持续运行 Agent）
- **内容**：A2A spec 假设 agent 持续在线监听，但大量真实 Agent 是 cron/heartbeat 模式（定时唤醒）
- **意义**：这正是 ACP 的主场！我们从设计上就支持：本地 relay 守护进程可长期运行，Agent 逻辑可定时唤醒，relay 代为缓冲消息（/recv 轮询）
- **ACP 优势**：`/recv` + `/stream` 双模式，天然支持 heartbeat agent 架构；A2A 还在讨论 `availability` 字段

#### #1463 — OID4VP for In-Task Authorization（去中心化授权）
- **内容**：OAuth 2.0 只能断言"有权限访问"，无法证明"具备某种能力/资质"（如 Agent 的 skills）
- **意义**：跟 ACP `QuerySkill()` 方向相关，但 OID4VP 更复杂（需 Verifiable Credentials 基础设施）
- **ACP 立场**：QuerySkill 直接 HTTP query，零基础设施，不需要 VC 体系
