### 重点 Issues（均为 2026-03-23 活跃）

#### #1667 心跳 Agent：可用性元数据 + 离线优先任务处理
**摘要**：A2A spec 假设 Agent 是持续在线的服务端。但现实中大量 Agent 是心跳式运行（每 4 小时 cron 唤醒），中间无服务器监听。问题：
- Caller 无法知道 Agent 是否在线
- `tasks/send` POST 到离线 Agent 直接失败
- 没有 queuing / offline-first 机制

**ACP 对比优势**：
- ACP 的 acp:// token 机制天然支持离线 Agent：Relay fallback 会 queue 消息到 Cloudflare Worker，Agent 上线后拉取
- ACP 的 `/recv` 端点（polled，持久化 JSONL）即是离线 Agent 的天然接收机制
- 这正是"个人/团队场景"vs"企业持续服务"的核心差异

**建议**：在 Show HN 和文档中强调 ACP 的离线优先特性，这是 A2A 的明显盲点。

---

#### #1672 Agent Card 身份验证提案
**摘要**：A2A 的 Agent Card 无法密码学验证真实性。当前依赖 transport 层信任（HTTPS/OAuth），覆盖授权但不覆盖身份。提议引入：
- Agent Card 中的 `verifiableCredential` 字段
- Ed25519 自签名声明
- DID 解析集成

**ACP 现状**：
- `did:acp:` 已在 v1.3 实现（Ed25519 key-based，自主权，无注册中心）
- HMAC-SHA256 签名已在 v1.1 实现
- **ACP 在这个方向已领先 A2A 约一个版本**

**建议**：在 spec 比较文档（docs/comparison.md）中加入这个维度对比。

---
