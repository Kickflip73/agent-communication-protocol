### 关键 Issue 跟踪

#### #1672 — Agent Identity Verification（2026-03-23 活跃，31 条评论）
**最新进展（03-23）：**
- 讨论向 **"混合模型"** 收敛：AgentCard 同时支持 CA 签发身份和自主主权身份
- 核心原则：**"Agent 声明自己的身份模型，验证方决定信任什么"**
- APS（Agent Passport System）的测试向量被引用，Ed25519 设计有参考价值
- ECDSA P-256（CA 签发）vs Ed25519（自主主权）仍在讨论，暂无定稿

**对 ACP 的影响：**
- ✅ 我们 v1.3 的 `did:acp:` 方向（自主主权 Ed25519）**被动吻合主流趋势**
- 可在 HN 发帖中强调：ACP 已在 v1.3 实现 did:acp: identity，A2A 仍在讨论
- ROADMAP v0.8 的 Ed25519 扩展节奏合理，无需加速

#### #1667 — Heartbeat-based Agents & Availability（2026-03-22 最新）
**最新进展（03-22）：**
- 讨论已进入"原型化"阶段，有人提出 `tasks/queue` 端点标准化提案
- 核心字段方案成型：`taskLatencyMaxSeconds`（SLA 约束）+ `nextActiveAt`（方便字段）
- 错误语义：若超出 bounded latency，应返回 `RESOURCE_EXHAUSTED` + 带 `retryAfter`

**对 ACP 的影响：**
- 这是 A2A 还没解决、ACP 也没完全解决的领域
- 短期：无需行动（不在 v1.4 scope）
- 中期（v1.0）：考虑在 AgentCard 中加入 `availability` 字段（已在能力声明中预留 `"availability": false`）

#### #1575 — APS Agent Identity 实现（03-05 最新，24 条评论）
- 讨论重点：委托链（delegation chain）+ 撤销（revocation）
- `cascadeRevoke()` 同步撤销整个子树，非最终一致性
- **结论：复杂度高，适合企业场景，ACP 不需要跟随**

#### #1627 — Error Code Mapping Fix（PR open，4 条评论）
- A2A 的 HTTP ↔ gRPC 错误码映射出现不一致（PR #1600 引入的问题）
- 修复方向：回滚 JSONRPC section 的修改，恢复 google.rpc.Code 对应关系
- **对 ACP 的影响：** ACP 使用自定义错误码（ERR_NOT_CONNECTED 等）+ HTTP 状态码，不依赖 gRPC 映射，**不受此影响**
