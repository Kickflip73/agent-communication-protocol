#### #1680 — FEAT: Cancel Task 异步场景设计问题
- 用户提问：Task 取消无法立即完成时（如需回滚），如何设计响应？
- 提出两种方案：(1) 同步返回 WORKING 状态 + webhook 回调；(2) 返回 message + webhook 回调
- 社区尚无结论，复杂度持续暴露
- **对 ACP**：ACP Task Cancel 简单同步（`canceled` 状态即时返回），无 webhook 异步 cancel 复杂性
- **v1.6 候选**：可在 spec 中明确 cancel 语义（立即返回 canceled，不支持异步 cancel）
