# ACP Minimal Agent Specification

**Version:** v0.6-dev  
**Status:** Draft  
**Date:** 2026-03-20  
**Authors:** J.A.R.V.I.S. / ACP Protocol Team

---

## Overview

This document defines the **minimum interface** a non-relay HTTP Agent must implement to participate in the ACP network. The goal: any existing FastAPI, Express, Flask, or Spring Boot service can become an ACP-compatible Agent by adding **3 endpoints**.

This is distinct from running `acp_relay.py` (the full relay daemon). Minimal Agents are lightweight peers — they receive messages, respond via SSE, and declare their capabilities via AgentCard.

---

## The 3 Required Endpoints

### 1. `GET /.well-known/acp.json` — AgentCard Discovery

Returns a JSON object describing who this Agent is and what it can do.

**Response (200 OK):**

```json
{
  "name": "my-agent",
  "version": "1.0.0",
  "acp_version": "0.6",
  "capabilities": {
    "streaming": true,
    "part_types": ["text"],
    "max_msg_bytes": 65536
  },
  "endpoints": {
    "send": "/message:send",
    "stream": "/stream",
    "agent_card": "/.well-known/acp.json"
  },
  "auth": {
    "schemes": ["none"]
  }
}
```

**Required fields:** `name`, `acp_version`, `capabilities`, `endpoints.send`  
**Optional:** `endpoints.stream`, `skills`, `auth`

---

### 2. `POST /message:send` — Receive Inbound Message

Called by a peer (or relay) to deliver a message to this Agent.

**Request body:**

```json
{
  "message_id": "msg_abc123",
  "role": "user",
  "parts": [
    {"type": "text", "content": "Hello, can you summarize this?"}
  ],
  "task_id": "task_xyz",
  "from": "peer-agent-name"
}
```

**Response (200 OK):**

```json
{
  "ok": true,
  "message_id": "msg_abc123"
}
```

**Idempotency:** If the same `message_id` is received twice, return `200 OK` without reprocessing (deduplicate on `message_id`).

**Error response:**

```json
{
  "ok": false,
  "error": "message too large",
  "failed_message_id": "msg_abc123"
}
```

---

### 3. `GET /stream` — Outbound SSE Stream *(optional but recommended)*

A Server-Sent Events (SSE) stream that pushes messages to the connected peer.

**Content-Type:** `text/event-stream`  
**Response format:**

```
data: {"type":"acp.message","message_id":"msg_reply1","server_seq":1,"ts":"2026-03-20T06:00:00Z","from":"my-agent","role":"agent","parts":[{"type":"text","content":"Here is the summary..."}]}\n\n

: keepalive\n\n
```

**Event types:**

| `type` | When | Required fields |
|--------|------|-----------------|
| `acp.message` | New message from this Agent | `message_id`, `parts`, `from`, `ts` |
| `acp.status` | Task state changed | `task_id`, `state` |
| `acp.artifact` | Produced output | `task_id`, `artifact` |
| `acp.peer` | Peer connected/disconnected | `event`, `peer_count` |

Send `keepalive` comments every ~30s to prevent connection timeouts.

---

## Part Model

Messages consist of one or more `Part` objects. Minimal Agents MUST support `text`; `file` and `data` are optional.

```json
{"type": "text",  "content": "plain text string"}
{"type": "file",  "url": "https://...", "media_type": "image/png", "filename": "diagram.png"}
{"type": "data",  "content": {"key": "value"}, "media_type": "application/json"}
```

---

## Connection Flow

When a Minimal Agent receives a link (e.g., `acp://IP:PORT/TOKEN`) from a peer, the flow is:

```
Minimal Agent                        ACP Relay Peer
      │                                    │
      │── GET /.well-known/acp.json ──────>│  (peer fetches card on connect)
      │<─ 200 AgentCard ──────────────────│
      │                                    │
      │<── POST /message:send ────────────│  (peer sends message)
      │─── 200 {"ok": true} ────────────>│
      │                                    │
      │── GET /stream ────────────────────>│  (Minimal Agent opens SSE to peer's /stream)
      │<── SSE events ─────────────────── │  (bidirectional via two SSE streams)
```

> **Note:** For bidirectional communication, both sides open SSE streams to each other. This avoids WebSocket complexity while remaining HTTP-native.

## Capability Declaration Reference

Full list of known `capabilities` fields:

| Field | Type | Default | Meaning |
|-------|------|---------|---------|
| `streaming` | bool | false | Supports SSE `/stream` endpoint |
| `push_notifications` | bool | false | Can push to webhooks |
| `input_required` | bool | false | Supports `input_required` task state |
| `multi_session` | bool | false | Supports concurrent peer connections |
| `query_skill` | bool | false | Supports `POST /skills/query` |
| `server_seq` | bool | false | Attaches `server_seq` to outbound messages |
| `part_types` | string[] | ["text"] | Supported Part types |
| `max_msg_bytes` | int | 65536 | Maximum message size in bytes |

## Minimal Implementation Example (Python/FastAPI)

```python
from fastapi import FastAPI, Request
from fastapi.responses import StreamingResponse, JSONResponse
import asyncio, uuid, json, datetime

app = FastAPI()
_inbox = asyncio.Queue()
_outbox = asyncio.Queue()

AGENT_CARD = {
    "name": "my-minimal-agent",
    "acp_version": "0.6",
    "capabilities": {"streaming": True, "part_types": ["text"], "max_msg_bytes": 65536},
    "endpoints": {"send": "/message:send", "stream": "/stream",
                  "agent_card": "/.well-known/acp.json"},
    "auth": {"schemes": ["none"]},
}

@app.get("/.well-known/acp.json")
async def agent_card():
    return AGENT_CARD

@app.post("/message:send")
async def receive_message(request: Request):
    body = await request.json()
    msg_id = body.get("message_id", str(uuid.uuid4()))
    await _inbox.put(body)
    # Echo reply via outbox
    reply = {
        "type": "acp.message",
        "message_id": "msg_" + uuid.uuid4().hex[:12],
        "server_seq": 1,
        "ts": datetime.datetime.utcnow().isoformat() + "Z",
        "from": "my-minimal-agent",
        "role": "agent",
        "parts": [{"type": "text", "content": f"Received: {body}"}],
    }
    await _outbox.put(reply)
    return {"ok": True, "message_id": msg_id}

@app.get("/stream")
async def stream():
    async def event_gen():
        while True:
            try:
                msg = await asyncio.wait_for(_outbox.get(), timeout=30)
                yield f"data: {json.dumps(msg)}\n\n"
            except asyncio.TimeoutError:
                yield ": keepalive\n\n"
    return StreamingResponse(event_gen(), media_type="text/event-stream")
```

This is a complete, functional ACP Minimal Agent in ~40 lines.

## Compatibility with acp_relay.py

A Minimal Agent can connect to a peer running `acp_relay.py` by:

1. Receiving the peer's `acp://IP:PORT/TOKEN` link
2. Fetching `GET http://IP:(PORT+100)/.well-known/acp.json` to exchange AgentCards
3. Posting messages to `POST http://IP:(PORT+100)/message:send`
4. Opening `GET http://IP:(PORT+100)/stream` as SSE listener

The relay peer handles all P2P/fallback transport complexity transparently.
