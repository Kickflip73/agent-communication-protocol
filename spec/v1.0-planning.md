# ACP v1.0 Planning

**Target:** 2026-05 (production release)  
**Theme:** Stability guarantee + production hardening  
**Based on:** [spec/core-v0.8.md](core-v0.8.md) + v0.9 changes

---

## Design Principles (unchanged from v0.9)

1. **Lightweight** — single-file relay, one command to run
2. **P2P, no middleman** — relay only punches holes; messages flow direct
3. **Practical** — any agent / framework / language; `curl`-accessible
4. **Personal & team scale** — contrast A2A's enterprise complexity

**Forbidden** (permanent red lines):
- ❌ OAuth 2.0 / PKCE
- ❌ Multi-tenancy
- ❌ gRPC
- ❌ Central registry / service discovery hub
- ❌ >5 Task states

### P1 — Ship with 1.0 (quality / completeness)

#### P1-1: Security Audit — HMAC implementation
- Review `_hmac_sign()` / `_hmac_verify()` in `acp_relay.py`
- Verify: constant-time comparison (`hmac.compare_digest`)
- Verify: no timing oracle in error responses
- Verify: replay protection (timestamp window check if applicable)
- Document findings in `docs/security.md`

#### P1-2: Security Audit — Ed25519 implementation
- Review `_ed25519_sign_msg()` / `_ed25519_verify_msg()`
- Verify: canonical form is deterministic (`sort_keys=True`)
- Verify: key file permissions enforced (`chmod 0600`)
- Verify: graceful fallback when `cryptography` not installed
- Document in `docs/security.md`

#### P1-3: `docs/security.md` — Security model document
- HMAC: what it protects (message integrity + peer auth), what it does NOT (confidentiality)
- Ed25519: identity binding, non-repudiation, key rotation procedure
- Transport security recommendations (TLS termination at load balancer)
- Known limitations: no forward secrecy, no encryption at rest

#### P1-4: Go SDK stub (`sdk/go/`)
- Minimal `RelayClient` struct with `Send()`, `Recv()`, `Stream()`
- Zero external dependencies (stdlib `net/http` + `bufio` for SSE)
- `go.mod` with module path `github.com/Kickflip73/agent-communication-protocol/sdk/go`
- README + 5 basic tests

### P2 — Nice to have (post-1.0 or 1.1)

#### P2-1: Rust SDK stub (`sdk/rust/`)
- `RelayClient` struct, async `tokio` + `reqwest`
- Publish to crates.io as `acp-relay-client`

#### P2-2: DID optional extension (`did:acp:`)
- `did:acp:<public_key_b58>` format
- Aligns with ANP direction, not forced
- Spec extension doc: `spec/did-v1.0.md`
- CLI: `--did` flag (requires Ed25519 identity)

#### P2-3: HTTP/2 transport binding
- Server push for SSE (replaces polling)
- Requires `hypercorn` or `h2` library (optional dep)
- Spec extension: `spec/transports.md` §HTTP2

#### P2-4: `relay/examples/` Docker + systemd
- `Dockerfile` for one-command relay deployment
- `relay.service` systemd unit for persistent background relay
