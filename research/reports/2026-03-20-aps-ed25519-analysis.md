## 1. APS Key Architecture (what to borrow)

### Key Format
- **Ed25519**: 32-byte raw keys, hex-encoded
- **Private key storage**: `pkcs8` DER format (48 bytes: 16-byte DER prefix + 32-byte raw key)
- **Public key storage**: `spki` DER format (44 bytes: 12-byte DER prefix + 32-byte raw key)
- DER prefix for private: `302e020100300506032b657004220420` (16 bytes)
- DER prefix for public: `302a300506032b6570032100` (12 bytes)

### Sign/Verify
```
sign(message: string, privateKeyHex) → signatureHex (128 hex chars = 64 bytes)
verify(message: string, signatureHex, publicKeyHex) → boolean
```

### AgentPassport data model
```typescript
{
  agentId: string          // unique agent identifier
  publicKey: string        // hex-encoded Ed25519 public key (64 chars)
  delegations: Delegation[]  // scoped capability grants
  signature: string        // self-signed by agent's private key
}
```

## 2. What ACP v0.8 Should Borrow

### Minimal Ed25519 identity for ACP (NOT full APS)

APS is 240-test, multi-module (delegation chains, reputation scores, governance, commerce...) — far too heavy for ACP's "Agent WhatsApp" positioning.

**What ACP needs (opt-in extension only):**

```python
# acp_relay.py additions (< 80 lines)

def generate_keypair() -> dict:
    """Generate Ed25519 keypair, stored in ~/.acp/identity.json"""
    from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PrivateKey
    priv = Ed25519PrivateKey.generate()
    pub = priv.public_key()
    return {
        "private_key": priv.private_bytes_raw().hex(),
        "public_key": pub.public_bytes_raw().hex()
    }

def sign_message(message_id: str, ts: str, private_key_hex: str) -> str:
    """Sign canonical message for ACP identity assertion"""
    from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PrivateKey
    canonical = f"{message_id}:{ts}"
    priv = Ed25519PrivateKey.from_private_bytes(bytes.fromhex(private_key_hex))
    return priv.sign(canonical.encode()).hex()

def verify_message(message_id: str, ts: str, sig_hex: str, public_key_hex: str) -> bool:
    """Verify Ed25519 signature on incoming message"""
    from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PublicKey
    from cryptography.exceptions import InvalidSignature
    canonical = f"{message_id}:{ts}"
    pub = Ed25519PublicKey.from_public_bytes(bytes.fromhex(public_key_hex))
    try:
        pub.verify(bytes.fromhex(sig_hex), canonical.encode())
        return True
    except InvalidSignature:
        return False
```

## 3. Key Differences: APS vs ACP Identity

| Dimension | APS (full) | ACP v0.8 (minimal opt-in) |
|-----------|-----------|--------------------------|
| Scope | Full identity + delegation + revocation + reputation | Message-level signing only |
| Key storage | PKCS8/SPKI DER | Raw 32-byte hex (simpler) |
| Delegation chains | Yes (cascade revocation) | No (not needed for P2P) |
| PKI / CA | No (self-signed) | No (self-signed, same approach) |
| Python lib | — | `cryptography` (pip install) |
| Mandatory | No (APS is opt-in SDK) | No (ACP `--identity` flag) |
| Lines of code | ~3,000 (TS) | ~80 (Python, added to acp_relay.py) |

## 4. Implementation Plan for ACP v0.8

**Phase 1: Key generation + storage**
- `--identity` flag: auto-generate keypair on first run, store in `~/.acp/identity.json`
- Expose `public_key` in AgentCard `identity` block
- `capabilities.ed25519_identity = true`

**Phase 2: Sign + attach**
- All outbound messages: append `sig_ed25519` field
- Canonical string: `{message_id}:{ts}` (same pattern as HMAC, consistent)

**Phase 3: Verify + trust**
- On inbound: if sender's AgentCard has `ed25519_identity`, verify `sig_ed25519`
- Mismatch: warn (same as HMAC — don't drop, don't block)
- No key exchange protocol needed: AgentCard public key is the trust anchor

**Dependencies:**
- Python `cryptography` library (already in many envs; pure-python fallback: `PyNaCl`)
- Fallback if unavailable: warn "ed25519 unavailable, install cryptography"

---

## 5. What NOT to Borrow from APS

- ❌ Delegation chains (cascade revocation) — overkill for P2P
- ❌ ReputationScore — not ACP's concern
- ❌ DER encoding complexity — use raw 32-byte keys directly
- ❌ Governance / voting weight — ACP is personal/team, not protocol council
- ❌ Mandatory PKI — self-signed is sufficient for "WhatsApp between agents"

## 6. Estimated Development Effort

~80 lines of Python code + unit tests. One heartbeat dev round.

**Dependency options (in order of preference):**
1. `cryptography` (PyCA, widely installed) — `Ed25519PrivateKey.generate()`
2. `PyNaCl` (libsodium binding) — `nacl.signing.SigningKey.generate()`
3. Fallback: stdio subprocess to `openssl` (zero Python deps, universal)
